Welcome to Bastion — the CIPP partner gateway

Resources on this page are reserved for the Tier 1 CSP partners we work with directly. Partner material is sealed with AES-256-GCM — it ships as ciphertext and is decrypted locally in your browser with the access code issued to your organisation. Your code never leaves this page.

Partner access

Unlock the partner vault

Enter the access code from your CyberDrain onboarding pack. Codes are issued per organisation and rotated when the partner roster changes.

🔏 Decryption happens entirely in your browser (WebCrypto: PBKDF2-SHA256 → AES-256-GCM). No code, cookie, or telemetry is sent anywhere.

Security

How Bastion is hardened

  • ✓ Sealed partner content — the vault is AES-256-GCM ciphertext with per-partner key slots (PBKDF2-SHA256, 600k iterations). Removing a partner's slot and resealing rotates the content key: revocation, not obfuscation.
  • ✓ Strict Content-Security-Policy — only the same-origin gate script may run; no inline script, no third-party origins, no external calls.
  • ✓ Fully self-contained — no CDNs, fonts, analytics, or trackers, so there is no third-party attack surface and nothing to leak.
  • ✓ No referrer leakage — referrer: no-referrer, rel="noopener" on outbound links, and noindex to keep the gateway out of search results.
  • ✓ Hardened deploy pipeline — least-privilege GitHub Actions with SHA-pinned dependencies, egress auditing, main-branch-only publishing, and a secrets guard that fails the build if partner material would leak. Deploy approval is enforced via the github-pages environment once required reviewers are configured (see setup).
  • ✓ Machine-readable security contact — see /.well-known/security.txt (RFC 9116).
